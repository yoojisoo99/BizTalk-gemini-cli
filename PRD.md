# PRD: BizTone Converter (업무 말투 변환기)

## 1. 개요 (Introduction)

- **제품명**: BizTone Converter - 업무 말투 자동 변환 솔루션
- **문제 정의**: 많은 직장인, 특히 신입사원이나 비즈니스 커뮤니케이션에 익숙하지 않은 구성원들은 대상(상사, 동료, 고객)과 상황에 맞는 적절한 업무용 말투를 사용하는 데 어려움을 겪습니다. 이는 커뮤니케이션의 비효율과 오해를 유발할 수 있으며, 개인과 조직의 전문성에 영향을 미칩니다.
- **제품 비전**: 누구나 쉽고 빠르게 자신의 생각을 전문적인 비즈니스 언어로 변환하여, 명확하고 효과적인 커뮤니케이션을 수행할 수 있도록 돕는 AI 기반의 웹 솔루션입니다.

---

## 2. 목표 (Goals & Objectives)

### 2.1. 비즈니스 목표
- **업무 생산성 향상**: 커뮤니케이션 문구 작성에 소요되는 시간을 단축하여 핵심 업무에 집중할 수 있도록 지원합니다.
- **커뮤니케이션 품질 표준화**: 조직 내외부 커뮤니케이션의 일관된 톤앤매너를 유지하여 기업 이미지를 제고합니다.
- **직원 교육 비용 절감**: 신입사원 및 주니어 직원의 비즈니스 작문 교육을 보조하는 효과적인 도구를 제공합니다.

### 2.2. 제품 목표
- **핵심 기능 제공**: 사용자의 일상적인 표현을 '상사', '동료', '고객' 대상에 맞는 전문적인 말투로 정확하게 변환합니다.
- **사용자 경험 최적화**: 직관적이고 반응성이 뛰어난 웹 인터페이스를 통해 모든 디바이스에서 쉽고 빠른 사용성을 제공합니다.
- **성능 및 안정성**: 평균 3초 이내의 빠른 변환 속도와 안정적인 서비스 가용성을 보장합니다.

### 2.3. 성공 지표 (Success Metrics)
- **주간 활성 사용자(WAU)**: 1,000명 달성
- **서비스 만족도**: 사용자 피드백 기능에서 "도움됨" 비율 85% 이상 유지
- **변환 속도**: API 요청부터 결과 반환까지 평균 3초 이내 응답률 95% 이상
- **리텐션**: 첫 사용 후 1주일 내 재방문율 30% 달성

## 3. 대상 고객 (Target Audience)

- **Primary Persona**: **신입사원/주니어 (1~3년차)**
  - **특징**: 업무 경험이 적어 비즈니스 용어 및 보고, 요청, 거절 등의 상황별 표현에 어려움을 겪음.
  - **Needs**: 정중하면서도 명확하게 자신의 의사를 전달하고 싶어함. 실수를 줄이고 싶어함.
- **Secondary Persona**: **타 부서/외부와의 소통이 잦은 중간 관리자**
  - **특징**: 다양한 이해관계자와의 커뮤니케이션으로 톤앤매너 유지가 중요함. 빠른 문서 작성이 필요함.
  - **Needs**: 상황에 맞는 최적의 표현을 빠르게 찾아 업무 효율을 높이고 싶어함.
- **Tertiary Persona**: **고객 응대 전문가 (영업/CS 담당자)**
  - **특징**: 공식적이고 정중한 표현을 일관되게 사용해야 함.
  - **Needs**: 고객에게 신뢰감을 주는 표준화된 커뮤니케이션 가이드가 필요함.

### 3.1. 수신자 페르소나 (변환 대상)
본 서비스는 메시지를 받는 상대를 아래와 같이 세 그룹으로 정의하고, 각 그룹에 최적화된 결과물을 제공합니다.

*   **상사 (Upward):**
    *   **특징:** 보고의 명확성, 격식, 신뢰성을 중시함.
    *   **결과물 톤앤매너:** 정중한 격식체. 결론부터 명확하게 제시하는 보고 형식.
*   **타팀 동료 (Lateral):**
    *   **특징:** 협업의 원활함, 요청의 명확성을 중시함.
    *   **결과물 톤앤매너:** 친절하고 상호 존중하는 어투. 요청 사항과 마감 기한을 명확히 전달하는 협조 요청 형식.
*   **고객 (External):**
    *   **특징:** 서비스의 신뢰도, 친절함, 문제 해결을 중시함.
    *   **결과물 톤앤매너:** 극존칭을 사용하며, 전문성과 서비스 마인드를 강조. 안내, 공지, 사과 등의 목적에 부합하는 형식.

## 5. 화면 디자인 및 스타일 가이드 (UI/UX Design & Style Guide)

### 5.1. 디자인 원칙
- **직관성 (Intuitive)**: 사용자가 별도의 학습 없이 기능을 즉시 파악하고 사용할 수 있도록 간결하고 명확한 인터페이스를 제공합니다.
- **전문성 (Professional)**: 신뢰감을 주는 깔끔하고 정돈된 비즈니스 톤의 디자인을 유지합니다.
- **효율성 (Efficient)**: 사용자가 최소한의 동작으로 '입력 → 변환 → 복사'의 핵심 플로우를 완료할 수 있도록 설계합니다.

### 5.2. 레이아웃
- **데스크톱**: 2단 분할 레이아웃을 적용하여 좌측에는 원문 입력창, 우측에는 변환 결과창을 배치합니다. 이를 통해 사용자는 원문과 변환된 텍스트를 한눈에 비교할 수 있습니다.
- **모바일/태블릿**: 화면 너비가 768px 이하일 경우, 2단 레이아웃이 세로 스택(상: 원문 입력, 하: 변환 결과) 형태로 자동 전환되는 반응형 레이아웃을 구현합니다.

### 5.3. 색상 팔레트 (Color Palette)
- **Primary Color**: `#4A90E2` (신뢰감을 주는 파란색)
  - 사용처: 변환 버튼, 활성화된 UI 요소, 링크 등
- **Secondary Color**: `#F5F7FA` (부드러운 연회색)
  - 사용처: 페이지 전체 배경
- **Text Color**: `#333333` (기본 텍스트), `#FFFFFF` (버튼 위 텍스트)
- **Accent & Feedback Color**:
  - `Success`: `#50E3C2` (복사 성공 등 긍정 피드백)
  - `Error`: `#D0021B` (API 오류 등 부정 피드백)
  - `Border`: `#E0E6ED` (입력창 테두리)

### 5.4. 타이포그래피 (Typography)
- **기본 글꼴**: `Pretendard`, `Noto Sans KR`, `sans-serif` (가독성이 높은 웹폰트 우선)
- **글꼴 크기**:
  - 제목 (h1): `28px`
  - 부제목 (h2): `22px`
  - 본문 (body): `16px`
  - 보조 텍스트 (caption): `14px`

### 5.5. 컴포넌트 스타일
- **버튼**:
  - 기본 스타일: Primary 색상 배경, 흰색 텍스트, 4px의 둥근 모서리(border-radius).
  - 호버(Hover) 효과: 마우스를 올리면 밝기가 10% 증가하는 미묘한 전환 효과.
  - 로딩 상태: 변환 중일 때 버튼 내부에 스피너(spinner) 애니메이션 표시.
- **입력창 (Textarea)**:
  - 명확한 테두리(Border)와 12px의 내부 여백(padding)을 가집니다.
  - 포커스(Focus) 시 테두리 색상이 Primary 색상으로 변경됩니다.
- **카드 (Card)**:
  - 입력창과 결과창은 시각적 구분을 위해 은은한 그림자(box-shadow) 효과가 적용된 카드 UI 내에 배치됩니다.

## 8. 프로젝트 구조 (Project Structure)

프로젝트는 모듈화된 접근 방식을 사용하여 프론트엔드와 백엔드 컴포넌트를 명확히 분리합니다.

- **`./` (프로젝트 루트)**:
    - `.env`: 환경 변수 설정 파일.
    - `.gitignore`: 버전 관리에서 제외할 파일 및 디렉토리 지정.
    - `PRD.md`: Product Requirements Document 문서.
    - `프로그램개요서.md`: 프로젝트 초기 기획 문서.
- **`backend/`**: Flask 기반의 백엔드 API 서버를 포함합니다.
    - `app.py`: Flask 애플리케이션의 메인 엔트리 포인트와 Groq AI API 연동 및 비즈니스 로직 처리.
    - `requirements.txt`: Python 패키지 의존성 목록.
- **`frontend/`**: 사용자 인터페이스(UI)를 구현하는 정적 웹 파일을 포함합니다.
    - `index.html`: 애플리케이션의 메인 HTML 파일.
    - `css/style.css`: 애플리케이션의 전반적인 스타일 시트.
    - `js/script.js`: 클라이언트 측 JavaScript 로직 (API 호출, DOM 조작 등).
- **`.venv/`**: Python 가상 환경 디렉토리 (개발 환경 격리).
