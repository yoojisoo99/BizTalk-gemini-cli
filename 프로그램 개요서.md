# 업무 말투 변환기 프로그램 (Business Tone Converter)

## 1. 프로젝트 개요
- **프로젝트 명**: BizTone Converter - 업무 말투 자동 변환 솔루션
- **개발 배경 및 필요성**:
  - 현대 업무 환경에서는 다양한 이해관계자(상사, 동료, 고객)에 따라 적절한 커뮤니케이션 방식이 요구됨
  - 신입사원이나 비즈니스 커뮤니케이션에 익숙하지 않은 직원들이 적절한 업무 말투를 사용하는 데 어려움을 겪음
  - 일관된 비즈니스 커뮤니케이션 품질 유지와 시간 절감을 위한 자동화 솔루션 필요
- **주요 목표**:
  1. 사용자의 일상적인 표현을 업무에 적합한 전문적인 말투로 변환
  2. 대상(상사/동료/고객)에 따른 맞춤형 변환 제공
  3. 직관적인 웹 인터페이스를 통한 쉬운 접근성 확보
  4. 빠른 응답 시간과 안정적인 서비스 제공
- **목표 사용자**:
  - 기업의 신입사원 및 주니어 직원
  - 비즈니스 커뮤니케이션에 어려움을 겪는 직원
  - 다수의 이해관계자와 소통해야 하는 관리자
  - 외부 고객과의 공식적인 커뮤니케이션이 필요한 영업/CS 담당자

## 2. 주요 기능
### 2.1 핵심 기능
- **대상별 맞춤형 변환**: 상사(경어, 존댓말), 타팀 동료(중립적 존댓말), 고객(공식적이고 정중한 어투)으로 구분
- **실시간 변환 처리**: 사용자 입력 후 즉시 변환 결과 제공 (평균 3초 이내 응답 목표)
- **변환 기록 비교**: 원본 텍스트와 변환된 텍스트를 나란히 비교하여 학습 효과 제공
- **결과 복사 기능**: 변환된 텍스트를 클립보드에 한 번에 복사할 수 있는 기능
- **반응형 디자인**: 데스크톱, 태블릿, 모바일 등 다양한 디바이스에서 최적화된 화면 제공

### 2.2 사용자 시나리오
1. **신입사원의 상사 보고 메시지 작성 시나리오**
   - 상황: 신입사원 A씨가 상사에게 진행 중인 프로젝트 지연 상황을 알려야 함
   - 문제: 어떻게 정중하게 전달해야 할지 모름
   - 해결: "프로젝트 좀 늦어질 것 같아요" 입력 → 상사 대상 선택 → "프로젝트 일정이 다소 지연될 예정입니다. 빠른 시일 내에 완료하도록 하겠습니다."로 변환

2. **타팀과의 협업 요청 시나리오**
   - 상황: B팀의 매니저가 C팀에 자료 요청을 해야 함
   - 문제: 적절한 수준의 공식성을 유지하면서도 우호적인 어조 필요
   - 해결: "이 자료 좀 보내줄 수 있어?" 입력 → 타팀 동료 대상 선택 → "협업을 위해 관련 자료 공유 부탁드립니다. 가능하시다면 오늘 중으로 보내주시면 감사하겠습니다."로 변환

## 3. 시스템 아키텍처
### 3.1 기술 스택
#### 프론트엔드
- **HTML5**: 시맨틱 태그를 활용한 구조적 마크업
- **CSS3**: Flexbox/Grid를 활용한 반응형 레이아웃, CSS 변수 활용 테마 관리
- **JavaScript (ES6+)**: Fetch API를 통한 비동기 통신, DOM 조작, 이벤트 처리

#### 백엔드
- **Python 3.11**: 안정적인 버전의 Python 환경
- **Flask 2.3.x**: 경량 웹 프레임워크, RESTful API 엔드포인트 제공
- **Flask-CORS**: 크로스 오리진 요청 처리
- **python-dotenv**: 환경 변수 관리

#### AI/ML 통합
- **Groq AI API**: Groq 모델을 활용한 자연어 변환 처리

#### 배포 환경
- **Git Repository (GitHub)**: 소스 코드 버전 관리 및 협업
- **Vercel**: 정적 파일 호스팅 및 서버리스 함수 배포 플랫폼

### 3.2 시스템 구성도
```
사용자 브라우저
↓
[HTML/CSS/JS 프론트엔드]
↓ (HTTP POST/GET)
[Flask 백엔드 API 서버]
↓ (API 호출)
[Groq AI API 서비스]
↓
[응답 처리 및 반환]
```
**구성 요소 설명**:
1. **클라이언트 레이어**: 반응형 웹 인터페이스 제공
2. **API 게이트웨이**: Flask 기반 REST API 서버
3. **AI 처리 레이어**: Groq API를 통한 자연어 변환
4. **배포 인프라**: Vercel을 통한 서버리스 아키텍처

### 3.3 데이터 흐름
1. **사용자 입력**: HTML 폼을 통해 원본 텍스트와 대상 선택값 수집
2. **프론트엔드 처리**: JavaScript를 사용한 입력값 유효성 검증 및 JSON 형식 직렬화
3. **API 요청**: Fetch API를 사용하여 Flask 엔드포인트로 POST 요청 전송
4. **백엔드 처리**: Flask 애플리케이션에서 Groq API 호출을 위한 파라미터 구성
5. **AI 변환**: Groq API에 프롬프트 전송 및 응답 수신
6. **응답 반환**: 변환된 텍스트를 JSON 형식으로 클라이언트에 반환
7. **결과 표시**: JavaScript를 통해 화면에 변환 결과 동적 렌더링

## 4. 상세 기능 설명
### 4.1 입력 인터페이스
- **직관적인 폼 디자인**: 단일 페이지에서 모든 입력과 결과 확인 가능
- **대상 선택 드롭다운**: '상사', '타팀 동료', '고객' 중 선택 (기본값: 상사)
- **텍스트 입력 영역**: 최대 500자 입력 가능, 실시간 글자 수 표시
- **입력 가이드**: 각 대상별 권장 입력 예시 도구팁 제공
- **접근성 고려**: ARIA 라벨, 키보드 네비게이션 지원

### 4.2 말투 변환 엔진
- **Groq API 프롬프트 엔지니어링**: 대상별 최적화된 시스템 프롬프트 설계
  - 상사: "다음 문장을 상사에게 적합한 존댓말과 경어를 사용하여 변환해주세요..."
  - 동료: "다음 문장을 타팀 동료에게 적합한 중립적이지만 예의바른 업무 말투로 변환해주세요..."
  - 고객: "다음 문장을 고객에게 적합한 공식적이고 정중한 비즈니스 어투로 변환해주세요..."
- **에러 처리**: API 호출 실패 시 사용자 친화적 오류 메시지 및 재시도 옵션 제공
- **로깅 시스템**: 변환 요청 로그 기록 (개인정보 제외)

### 4.3 출력 인터페이스
- **이중 패널 레이아웃**: 좌측-원본 텍스트 / 우측-변환 결과 시각적 구분
- **형태소 강조 표시**: 변환 과정에서 주요 변경점 하이라이트
- **결과 복사 버튼**: 클립보드 복사 기능과 성공/실패 피드백
- **추가 옵션**: 변환 결과에 대한 피드백 수집 기능 (도움됨/도움안됨)
- **반응형 표시**: 모바일에서 세로 스택 레이아웃으로 자동 전환

## 5. 개발 및 배포 계획
### 5.1 개발 단계
1. **1단계: 기본 UI 구현**
   - HTML5 시맨틱 구조 설계
   - CSS3를 이용한 반응형 레이아웃 구현
   - JavaScript 기본 인터랙션 개발
   - 로컬 스토리지를 이용한 입력값 임시 저장 기능

2. **2단계: Flask 백엔드 서버 구축**
   - Flask 애플리케이션 기본 구조 구성
   - REST API 엔드포인트 설계 (`/api/convert`, `/health` 등)
   - CORS 설정 및 기본 에러 핸들링 구현
   - 로컬 개발 서버 설정

3. **3단계: Groq AI API 연동 구현**
   - Groq API 인증 및 호출 로직 구현
   - 대상별 프롬프트 템플릿 최적화
   - API 응답 파싱 및 에러 처리 로직 완성
   - 요청 제한 및 속도 제한 관리

4. **4단계: 통합 테스트 및 디버깅**
   - 프론트엔드-백엔드 통합 테스트
   - 다양한 입력값에 대한 테스트 케이스 작성

5. **5단계: Vercel 배포 및 최적화**
   - Vercel 프로젝트 설정
   - 환경 변수 구성 (API 키 보안 관리)

### 5.2 배포 전략
- **Git 버전 관리 전략**:
  - main(production), develop, feature 브랜치 전략 적용
  - Pull Request를 통한 코드 리뷰 필수화
  - 의미 있는 커밋 메시지 컨벤션 적용

- **Vercel CI/CD 자동화**:
  - GitHub 리포지토리와 Vercel 프로젝트 연동
  - main 브랜치 푸시 시 자동 배포 트리거
  - 프리뷰 배포를 통한 개발 중 테스트 환경 제공
  - 환경별(production, preview) 변수 분리 관리

- **환경 변수 관리**:
  - Groq API 키는 Vercel 환경 변수로 관리
  - 프론트엔드에서 접근하지 못하도록 서버사이드만 접근 가능 설정
